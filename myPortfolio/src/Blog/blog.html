One thing I learned in class today is about the importance of Git and Github. They can be tools for programmers to code as a team effectively, which is instrumental for tackling huge projects. This is accomplished by storing code on Github, and allowing people working together on a project while tracking the changes being made. A great workflow to follow when creating a webpage is to plan out the layout by drawing it out. Then you should take the drawing and plan out which way the HTML should be coded. Once you complete what text and images should be included, you can use CSS to improve the aesthetic and look of the webpage. Finally, coding with JavaScript will add interactivity features to the site. The Web Accessibility Initiative’s Accessible Rich Internet Applications specification (WAI-ARIA, or just ARIA) is good for bridging areas with accessibility issues that can’t be managed with native HTML. A screenreader is an internet tool which allows for people who are blind or visually impaired to be able to interpret what is on a computer screen by reading aloud what the user is browsing. Sometimes HTML cannot be used to specify the semantics, or meaning, of a webpage. ARIA, or Web Accessibility Initiative’s Accessible Rich Internet Applications specification, allows one to specify semantics by modifying the way an element is translated.
